# ورودی کاربران در bash

خب تو بخش قبلی یاد گرفتیم چجوری متغیر بسازیم و چاپشون کنیم. همینطور یاد گرفتیم پارامتر های ورودی رو هم بگیریم که کار آسونی بود.

حالا بریم یکم جالب ترش کنیم. شاید دلمون بخواد از کاربر سوال بپرسیم و مقدار کاربر رو دریافت کنیم! پس بریم شروع کنیم، بازم مثل همیشه اون فایل first.sh رو باز کنید و اینو توش بریزید:

```bash
#!/bin/bash

echo "What is your name?"
read name

echo "Hi there $name"
echo "Welcome to programming world!"
```

وقتی کد بالا رو اجرا بگیرید، اول یه سوال میپرسه و توی خط بعدی منتظر میمونه تا کاربر متن یا رشته (string) خودش رو وارد کنه و بعد اینتر (Enter) بزنه. به همین سادگی مقدار کاربر در متغیر name ذخیره میشه.

و در آخر هم اون متغیر رو خیلی ساده و خوشمزه چاپ میکنیم (آپولو هوا کردیم)


اجرا و خروجی این شکلی میشه تقریبا:

* اجرا کردن اسکریپت زیبامون:

```bash
./first.sh
```

* بعدش اسممونو مینویسم و اینتر میزنیم:

```
What is your name?
Dani
```

* در آخرم خروجی رو میبینیم و عشق میکنیم:

```
Hi there Dani
Welcome to programming world!
```
بیاید کد بالا رو یکم بهترش کنیم. شاید بخوایم دقیقا روبه روی متنمون از کاربر ورودی بگیریم و نره خط بعدی! اونوقت میایم از فلگ p استفاده میکنیم برای read به اینصورت read -p. این فلگ کارمونو خیلی ساده میکنه و دیگه نیازی به echo کردن نداریم. در واقع این فلگ باعث میشه ما بتونیم متن خودمونو بزاریم و ورودی رو از کاربر بگیریم، بدون استفاده از چیز دیگه ایی:

```bash
#!/bin/bash

read -p "What is your name? " name

echo "Hi there $name"
echo "Welcome to programming world!"
```

دیگه اینو خودتون تست کنید و ببینید فرقش چیه! خدایی راحت تره و بهتره...
