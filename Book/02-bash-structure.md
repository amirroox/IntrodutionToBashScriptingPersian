# ساختار BASH

به طور کلی انتظار دارم با لینوکس کار کرده باشید یکم و دستورات مهمش رو بدونید؛ اما بازم سعی میکنم تا جایی که میشه ساده بگم تا قابل درک باشه. 

همین اول کار برای ایجاد اسکریپت باید یه فایل بسازیم! پسوند فایل های بَش .sh هستن. مثلا یه فایل میسازیم به نام first.sh. 

توی لینوکس برای ایجاد فایل از دستور touch استفاده میکنیم (که البته هدف اصلیش برای تغییر زمان فایله)

```bash
touch first.sh
```

روش های دیگه ایی هم برای ایجاد فایل هست که چنتاشو این زیر میزارم

```bash
nano first.sh  # With Editor
echo '' > first.sh  # With Echo
```

یه مورد خیلی مهم که باید ذکر بشه! برای اجرا کردن این فایلی که نوشتیم باید اول فایلتون نشون بدیم که این فایل باید با چی باز بشه!

```bash
# اول فایلتون بزارید
#!/bin/bash
```

به این میگن [Shebang](https://en.wikipedia.org/wiki/Shebang_(Unix)).

کل کاریم که شی بانگ میکنه اینه که به سیستم عامل دستور میده که این فایلی که من نوشتم رو با bash اجرا کنه. (چیزی که توی فایل نوشتیم هم مسیر اجراییشه)

همونطور که بالا گفتم، چیزی که نوشتیم مسیر اجراییه پس ممکنه توی بعضی از سیستم عامل ها مسیر درستی نباشه! مخصوصا تو سیستم عامل های غیر لینوکسی. پس اگه مسیر بالا رو ندارید شاید مسیر پایین بدرتون خورد :)

```bash
#!/usr/bin/env bash
```
دستور بالا میاید توی متغیر PATH میگرده و اگه فایل اجرایی bash رو پیدا کرد با اون اجراش میکنه. اگر بازم مشکل داشتید دیگه خودتون بگردید ببینید بَشِتون کجاس!
