# آرگومان ها در bash

همونجور که قبلا دیدیم، میتونیم به فایلمون آرگومان ورودی پاس بدیم. فرضا یه ابزار نوشتیم که که میخوایم ورودی اولمون یه کاری کنه و نمیخوایم بعد از اجرا ازمون بپرسه (یعنی خیلی راحت وقتی فایل رو اجرا میگیریم اون ورودی رو پاس میدیم به برنامه) برای مثال:

```bash
./fisrt.sh your_argument
```

بازم همونطور که قبلا هم اشاره کردیم، برای اینکه بتونیم به این آرگومان های ورودی دسترسی داشته باشیم باید از علامت `$` استفاده کنیم. یعنی برای آرگومان اول میشه `$1` و برای آرگومان دوم میشه `$2` همینجوری الی آخر.

میتونید رو همون فایل قبلی تست کنید، یا اینکه برید یه فایل جدید بسازید و رو اون تست کنید:

```bash
#!/bin/bash

echo "Argument one is $1"
echo "Argument two is $2"
echo "Argument three is $3"
```

فرضا اگه فایل بالا رو با 3 تا آرگومان اجرا بگیریم:

```bash
./fisrt.sh dog cat bird
```

خروجی میشه این:

```bash
Argument one is dog
Argument two is cat
Argument three is bird
```

همچنین توجه داشته باشید که اگه 3 آرگومان پاس ندیم و فرضا فقط یکی بدیم، چون آرگومان دوم و سوم خالی میمونه، پس مقدارشم خالیه!

یادتون باشه اگه خواستید به همه آرگومان ها همزمان دسترسی داشته باشید میتونید از `$@` استفاده کنید:

```bash
#!/bin/bash

echo "All arguments: $@"
```

بازم همون دستور قبلی رو میزنیم تا تست کنیم:

```bash
./fisrt.sh dog cat bird
```

حالا خروجی میشه این:

```
All arguments: dog cat bird
```

یه چیزی که باید یادتون باشه اینه که توی برنامه نویسی همه چیز از 0 شروع میشه، پس اونوقت `$0` چه معنایی داره؟ 

اگه درست حدس زده باشید، `$0` میشه دقیقا همون فایل اجرایی تون. برای تست کردن و فهمیدن این رو بنویسید:

```bash
#!/bin/bash

echo "$0 $1 $2 $3";
```

و اینجوری اجراش کنید:

```bash
./first.sh 1 2 3
```

خودتون متوجه میشید که `$0` دقیقا میشه چی!

شاید بپرسید اصلا به چه کاری میاد گرفتن اسم فایل اجرایمون! باید بگم یه روش عالی برای ساخت اسکریپت خود تخریب گره! چی هست؟ شاید بخواید اسکریپت بعد از اجرا خودش رو پاک کنه (ویرووووس)، همچنین اسم فایلتونم میده که شاید جایی بدرتون بخوره!

بیاید یه مثال ساده بزنیم از اسکریپتی که یه کاری میکنه و بعد خودش رو پاک میکنه، البته اصلا ویروس نیست و یه چیز خیلی سادست:

```bash
#!/bin/bash

echo "The name of the file is: $0 and it is going to be self-deleted."

rm -f $0
```

اینو اجرا بگیرید و ببینید که بعد از اجرا، فایلتون پاک میشه! فقط توجه کنید که اگه روی ابزار خاصی کار میکنید، همیشه ازش یه نسخه پشتیبان بگیرید که ناناحت نشید.

همچنین یه دستور داریم بالا به نام `rm -f` که احتمالا بلد باشید چیه، اما بازم میگم. این دستور برای پاک کردن فایل استفاده میشه و فلگ `f` مخفف force عه که یعنی حتما پاکش کن! که برای همین باید مراقب این دستور باشید، چون وقتی چیزی پاک شد دیگه پاک شده!
